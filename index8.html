Selectores avanzados
-"," (coma): aplica a todos los elementos, agrupa varios elementos comparten una serie de declaraciones iguales. Ej: p, a, div {}. mas info
-" " (espacio): aplica a los selectores descendientes, apunta a elementos contenidos dentro de otro en la estructura del documento. Ej: div p {}. mas info
-">" (mayor): aplica a los selectores hijos directos, selecciona los elementos que sean hijos directos del contenedor padre, descartando nietos y sucesivos. Ej: span > a {}. mas info
-"+" (signo mas):aplica a un hermano adyacente, para seleccionar un elemento que está directamente después de otro elemento específico, deben tener el mismo elemento padre . Ej: div + p {}. mas info
-"~" (virgulilla):aplica general de hermanos, selecciona todos los elementos que son los siguientes hermanos de un elemento específico. Ej: div ~ p {}. mas info

Pseudoclases
Pseudoclase es un selector que marca los elementos que están en un estado específico o tienen un comportamiento determinado.
- :first-child = modifica el estilo del primer elemento de un grupo de elementos hermanos dentro de un contenedor, es decir “el primer hijo de su padre”. Ej: p:first-child {}. mas info
- :last-child = igual que el anterior pero al reves, es el último hijo de su padre.Ej: p:last-child {}. mas info
- :nth-child(n) = El selector coincide con cada elemento que es el n-ésimo hijo, independientemente del tipo, de su padre. n puede ser un número, una palabra clave o una fórmula. Ej: p:nth-child(2) {}. mas info

Pseudoelementos

Transformaciones, animaciones y transiciones
MAS INFO ACA
Permiten mover, rotar, escalar y sesgar elementos, es decir, efectos visuales en 2D y 3D.
- transform = aplica una o varias funciones de transf. sobre 1 elemento.
- transform-origin = cambia punto de origen del elemento en una transformacion.
- transform-style = cambia el tratamiento de los elementos hijos.

2D - Traslaciones (translate)
Realizan una transformación en la que mueven un elemento de un lugar a otro.Si especificamos un valor positivo en el eje X (horizontal), lo moveremos hacia la derecha, y si especificamos un valor negativo, lo moveremos hacia la izquierda. Lo mismo con el eje Y (vertical). mas info aca
- translateX(x) = mueve el elemento una distancia de size x, horizontalmente
- translateY(y) = mueve el elemento una distancia de size y, verticalmente
- translate(x, y) = propiedad de atajo de las dos anteriores.

2D - Escalado (scale)
Realizan una transformación en la que aumentan o reducen el tamaño de un elemento, basándose en el parámetro indicado, que no es más que un factor de escala:
- scaleX(fx) = reescala el elemento a un nuevo tamaño con un factor number horizontalmente
- scaleY(fy) = reescala el elemento a un nuevo tamaño con un factor number verticalmente
- scale(fx, fy) = propiedad de atajo de las dos anteriores.
Si utilizamos scale() con dos parámetros iguales, estamos manteniendo la proporción del elemento, pero si utilizamos diferentes valores, acabaría deformandose. mas info aca

2D - Rotaciones (rotate)
Giran el elemento el número de grados indicado: mas info aca
- rotateX(xdeg) = rotacion 2d en direction solo para el eje horizontal
- rotateY(ydeg) = rotacion 2d en direction solo para el eje vertical
- rotate(deg) = rotacion 2d en direction sobre si mismo.

2D - Deformaciones (skew)
Establecen un ángulo para torcer, tumbar o inclinar un elemento en 2D.: mas info aca
- skewX(xdeg) = establece un angulo de direction para una deformacion 2d respecto al eje horizontal
- skewY(ydeg) = establece un angulo de direction para una deformacion 2d respecto al eje vertical

Funciones 3D
A las funciones anteriores, también podemos añadir las funciones para hacer uso del eje Z y con esto acceder a las dimensiones espaciales o “3D”. Las propiedades de transformación 3D son las siguientes: mas info aca
- translateZ(z) = traslada el elemento una distancia de size en el eje de la profundidad (z).
- translate3d(x, y, z) = establece una translacion 3D, donde aplica los parametros de size en cada eje.
- scaleZ(fz) = reescala el elemento a un nuevo tamaño con factor numer del eje z.
- scale3d(fx, fy, fz) = establece una escala 3D donde aplica los factores a cada eje.
- rotateZ(zdeg) = establece una rotacion 2D en direction zdeg grados solo para el eje de Z.
- rotate3d(x, y, z, deg) = establece una rotacion 3D aplicando un vector (z, y, z) y el angulo en direction deg.
- perspective(n) = establece una perspectiva 3D de size n.
- matrix3d(n, n, ...) = establece una matriz de transformacion 3D (16 valores).

Transformaciones múltiples (atajos)
Para no pisar transformaciones en el mismo elemento usamos, múltiples transformaciones, separándolas mediante un espacio. En el siguiente ejemplo, aplicamos una función de rotación, una función de escalado y una función de traslación de forma simultánea:
div { transform: rotate(5deg) scale(2,2) translate(20px, 40px); }

Transiciones
Permiten cambiar los valores de una propiedad durante un tiempo determinado. Es un cambio de estilo con un efecto suavizado entre un estado inicial y un estado final. Se necesida el dato del tipo de estilo al q quiero cambiar y por cuanto tiempo. mas info aca

- transition-property = Indica la propiedad que será afectada por la transición. Puede ser una propiedad concreta (width o color, por ej), all para que se aplique a todos los elementos, o none (ninguna transición).

- transition-duration = 0/time. Establece la duración de la transición. Se recomienda comenzar con valores cortos, para que las transiciones sean rápidas y elegantes. Una duración demasiado grande producirá una transición con detenciones intermitentes, y pueden resultar molestas a muchos usuarios.

- transition--timing-function = indica el ritmo de la transición que queremos conseguir.
Se recomienda comenzar con linear y luego utilizar otros valores para variar el ritmo sea al inicio y/o al final de la transición.
Los valores que puede tomar la propiedad son:
-ease (lento, rapido, lento), siempre es constante
-linear (normal, normal, normal), comienza suavemente, aumenta la velocidad y finaliza suavemente.
-ease-in (lento, normal, normal), son variaciones que van más lento al principio o al final
-ease-out (normal, normal, lento), son variaciones que van más lento al principio o al final
-ease-in-out (lento, normal, lento), una mezcla de ambas anteriores
-cubic-bezier(personalizado), más detalle la transición. cubic-bezier(A (x1), B(y1), C(x2), D(y2)), mas info aca

- transition-delay = retrasar el inicio de la transición los segundos especificados.

Atajo: se puede poner todo junto siguiento este orden usando la propiedad transition. mas info aca
ej:
div {
transition: all(property) 0.2s (duration) ease-in (timing-function)(delay);
}

Animaciones
mas info aca
Permite que un elemento cambie gradualmente de un estilo a otro, esto será posible de forma automática, sin que el usuario tenga que realizar una acción concreta.
Primero debemos especificar algunos fotogramas clave (@keyframes) para la animación, que contendrán los estilos que tendrá el elemento en determinados momentos. Por defecto, cuando se termina una animación que se ha indicado que se reproduzca sólo una vez, la animación vuelve a su estado inicial (primer fotograma).
Debemos ser cuidadosos al indicar el tiempo en las propiedades de duración. Al ser una unidad diferente a las que solemos manejar (px, em, etc...) hay que especificar siempre la s, por segundos, aunque sea un valor igual a 0.
Propiedades de las animaciones:
- animation-name = permite especificar el nombre del fotograma a utilizar. - animation-duration = 0/time. igual que transiciones. Nota: Si animation-duration no se especifica la propiedad, no se producirá ninguna animación, porque el valor predeterminado es 0s (0 segundos).
- animation-timing-fuction = ease/linear/ease-in/ease-out/ease-in-out/cubic, igual que transiciones
- animation--delay = 0/time, igual que transiciones. También se permiten valores negativos. Si usa valores negativos, la animación comenzará como si ya se hubiera estado reproduciendo durante N segundos.
- animation-iteration-count = permite indicar el número de veces que se repite la animación, o infinite para que se repita continuamente.
- animation-direction = indicar el orden en el que se reproducen los fotogramas, pudiendo escoger un valor de los siguientes:
*normal: desde el principio al final
*reverse: desde el final al principio
*alternate: La animación se reproduce primero hacia delante y luego hacia atrás.
*alternate-reverse: La animación se reproduce primero hacia atrás y luego hacia adelante.
- animation-fill-mode = qué debe mostrar la animación cuando ha finalizado y ya no se está reproduciendo, si mostrar el estado inicial (backwards), el estado final (forwards) o una combinación de ambas (both).
- animation-play-state = establecer la animación a estado de reproducción (running) o pausarla (paused).

atajo:todo en uno, mas info aca ej:
div {
animation: changeColor(nombre), 5s (duracion), linear (timing-function), 0.5s (delay), 4 (iteration-count), normal (direccion), forwards running (fill-mode)(play-state)
}

Fotogramas (keyframes: cuadros de animacion)
Para definir los fotogramas de una animación. Para que una animación enchulada funcione, debe vincular la animación a un elemento, para eso se usa keyframes.
Los selectores from y to son realmente sinónimos de 0% y 100%. También es posible utilizar porcentaje. Al usar el porcentaje, puede agregar tantos cambios de estilo como desee. Al modificarlos podremos ir añadiendo nuevos fotogramas intermedios:
@keyframes nombre {
selectorkeyframes{
propiedad: valor;
propiedad: valor;
} }
Ejemplo:
@keyframes cambiarColor {
from {background-color: red;}
to {background-color: yellow;}
}

Encadenar animaciones, (atajo):
Es posible encadenar múltiples animaciones, separando con comas las animaciones individuales y estableciendo un tiempo de retardo a cada animación posterior: mas info aca
.animated {
animation: moveRight 5s linear 0, /* Comienza a los 0s */
lookUp 2.5s linear 5s, /* Comienza a los 5s */
moveLeft 2s linear 7.5s, /* Comienza a los 7.5s (5 + 2.5) */
dissapear 2s linear 9.5s; /* Comienza a los 9.5s (2 + 7.5) */
}

Responsive
Una web responsive es aquella que es capaz de adaptarse a cualquier dispositivo, ya sean computadoras, portátiles, tablets o smartphones.
El diseño web responsivo adapta la estructura y los diferentes elementos de cada página web a las dimensiones y características de cada pantalla. Por otro lado, el diseño web adaptativo es menos flexible, y se basa en el uso de tamaños y características pre-establecidas.
A tener en cuenta:

-Flujo(The Flow): es cuando una pantalla se vuelve más pequeña, el contenido comienza a crecer verticalmente ocupando más espacio, y el contenido que se encuentra debajo va a ser desplazado hacia abajo. Puede ser:
-Adaptativo: lo que vamos a ver la clase que viene.
-Estatico: flujo de elementos no se desplaza, no se adapta al ancho del viewport y se pierde contenido o cierto contenido tapa a otro.

-Unidades de medida: Estaticas (pixel) y relativas (%)

-Valores minimos y maximos: Ventajas y desventajas de usar:
-"width o height : n%"": Si width se establece en un porcentaje y height se establece en "automático", la imagen responderá y se escalará hacia arriba y hacia abajo.
-max-width o max-height: En donde la imagen o elemento se reducirá si es necesario, pero nunca se escalará para ser más grande que su tamaño original.

-Puntos de control(Breakpoints): Es donde se quiebra el diseño, se crean con los media queries, que nos permiten determinar que si el mínimo del ancho de la pantalla tiene un valor en lugar de otro.

-Objetos anidados(Nested Objects): Idea de usar contenedores para moverlos mejor

-Mobile primero, segundo Desktop. Estadísticamente, los dispositivos móviles son mayoritariamente los que acceden a los sitios Web. Los dispositivos de escritorio tienden a utilizarse cada vez menos.

-System Font(Fuentes de la Web): son descargadas por lo que, cuantas más haya, más lento cargará el sitio.) vs WebFonts(Fuentes del Sistema): más rápidas, pero si NO están en el cliente navegador del usuario se usa una por defecto.)
Cuando estamos trabajando con dispositivos móviles tenemos que tener en cuenta que todo se carga.

-Bitmaps(JPG, PNG, GIF.): Recomendadas para muchos detalles y efectos. vs Vectors(SVG (gráficos basados en vectores escalables)): si voy a mostrar un ícono uso Icon Fonts, que son más livianos, pero algunos exploradores viejos no los soportan.

-Texto responsivo: El tamaño del texto se puede configurar con una unidad "vw", que es el "ancho de la ventana gráfica". De esa forma, el tamaño del texto seguirá el tamaño de la ventana del navegador. Ej: style = " font-size: 10vw "

Viewport es el tamaño de la ventana del navegador.


1vw = 1% del ancho de la ventana gráfica.

-Imágenes responsivas: con %, vw u etiquetas de tamaño de pantalla con sus minimos y maximos (width, height, min-width, min-height, max-height y max-width).
*background-size: puede ser:
-contain: la imagen de fondo se escalará e intentará ajustarse al área de contenido, pero mantendrá su relación proporcional entre el ancho y el alto de la imagen, como max-width.
-% (porcentaje): se extenderá para cubrir toda el área de contenido, se puede poner % en x y y, ej: (100%,100$).
-cover: es posible que se recorte parte de la imagen de fondo pero no se pixelea.
(dato?) Imagenes de fondo: < picture >(picture: También debe definir un elemento para navegadores que no admitan el < picture >elemento).

-Display(Tipos de caja): Cada elemento tiene un valor de display por defecto, los navegadores le dan a los elementos valores por defecto block e inline: -Block: El elemento empieza en una nueva línea (div, h1-h6, header, etc)
-Inline: Puede contener algo de texto dentro de un párrafo sin interrumpir el flujo del párrafo. (span, b, a)
-None: Oculta elementos sin eliminarlos, no deja un espacio donde el elemento se encontraba.
-Visibility: Igual que NONE pero mantiene un vacío con el mismo tamaño de lo que antes estaba ahí. Por defecto es visible pero puede ser: visible/hidden/collapse:solo para tablas.
-----Utilizar visibility: hidden es muy interesante si queremos que un elemento y su contenido se vuelva invisible, pero siga ocupando su espacio y así evitar que los elementos adyacentes se desplacen, lo que suele ser un comportamiento no deseado en algunas ocasiones cuando se aplica display: none.-----
-----Otra opción interesante es utilizar la propiedad opacity junto a transiciones o animaciones, desplazarse desde el valor 0 al 1 o viceversa. De esta forma conseguimos una animación de aparición o desvanecimiento.------
